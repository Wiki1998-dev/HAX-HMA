# Phase 2 Summary: Gradient SHAP on SNAX Cluster

**Status**: COMPLETE, VERIFIED
**Date**: 2026-04-07
**Project**: XAI on Heterogeneous Multi-Accelerator Platforms

## Executive Summary

Phase 2 successfully implemented and optimized the **Gradient SHAP** explainability method on the SNAX RISC-V cluster. The implementation was verified against a Python reference and achieved significant performance improvements through architectural optimizations.

- **Phase 2a (Baseline)**: 175,510 cycles (10,969 per sample)
- **Phase 2b (Optimized)**: 58,022 cycles (3,626 per sample)
- **Net Speedup**: **3.02×**
- **Correctness**: 0/256 BIST errors (max_err=0.000000)

## Implementation Details

### Model Configuration
- **Architecture**: Global Average Pooling (GAP) followed by a Fully Connected (FC) layer.
- **Dimensions**: H=4, W=4, K=16 (feature map) → C=10 (classes).
- **Data Type**: float32 (RISC-V FPU).
- **Samples**: N=16 baseline samples.

### Phase 2a: Sequential Dispatch
The initial implementation performed the full Gradient SHAP pipeline for every sample:
1. Interpolation between input and baseline.
2. Forward pass through GAP and FC.
3. Backward pass to compute gradients.
4. Accumulation of attribution values.

### Phase 2b: Architectural Optimization
Leveraging the mathematical properties of linear models (GAP+FC), we applied **Backward Hoisting**:
- Since the gradient is constant for linear layers, the forward pass and interpolation were eliminated.
- The backward pass was hoisted out of the sample loop and executed only once.
- The sample loop was reduced to a single, tight accumulation operation.

## Results Analysis

### Performance Comparison

|-------|-------------------|-------------------|--------|
| Interpolation | 48,702 | 0 | ELIMINATED |
| Forward Pass | 40,199 | 0 | ELIMINATED |
| Backward Pass | 30,341 | 1,930 | HOISTED |

### Bottleneck Identification
In the optimized Phase 2b kernel, **Accumulation** accounts for **91.6%** of total cycles. This stage is primarily memory-bound, involving two loads (input, baseline) and one fused multiply-add per element. Further acceleration will require multi-core parallelization or specialized vector units.
